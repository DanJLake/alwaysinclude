alwaysInclude

Stadardised and Accessible HTML guide.

The aim of this guide is to list all HTML5 tags and their attributes that should be included in almost any HTML file, for either accessibility reasons, or to adhere to web standards.

<!DOCTYPE html>

The doctype declaration is required as a DTD (Document Type Definition). <!DOCTYPE html> refers to HTML5.

<html>

The <html> tag tells browsers this document is written in HTML format.

The <html> tag should contain the following:

  • lang="" - This attribute tells the browser the language the content of the page is written in. For example, 'en' would be the value for english, however this can be extended to 'en-gb' or 'en-us'.

<link>

Links to external files such as CSS files and icons.

The <link> tag must always contain the following attributes:

  • href="" - the location of the file being linked. The file type extension must be included.
  • rel="" - specifies the contents of the document being linked. Possible values would be 'stylesheet' for a CSS stylesheet, or 'shortcut icon' for a favicon.

<script>

<header>

A <div> like container specifically for content placed at the top of a webpage, usually a top bar with a logo and navigation. A screen reader will recognise this as such allowing for greater accessibility.

<nav>

A container for the page navigation. Can be used by screen readers to quickly jump to the navigation links on a page. Usually placed inside the <header>, however it can be placed anywhere on the page.

<main>

The container that should hold the main content of a webpage. Screen readers will identify this content as the main content and allow the user to easily access it.

<article>

The container is used to wrap independent, self-contained content. Perfect for blog or news articles, or forum posts.

<footer>

The container that marks the page footer, this usually contains secondary navigation links, legal information, social media icons and other content that may be generalised site-wide, or not directly related to the content in the <main> section.

<section>

A container used to group related content. If none of the above HTML5 containers describe the section, or there are many sections on the page, this container allows screen readers to recognise the content inside is related.

A section tag should always contain the following:

  • id="" - An ID attribute identifies the nature and purpose of the section, whether vague or specific.

<figure>

<figcaption>

<img>

Image tags must always contain the following:

  • src="" - The src of an image is its source file. This can be a link to a file stored in any accessible location, whether its local (stored on the same server/in the same folder or directory as the html file), or located elsewhere online.
  • alt="" - The alt value of an image is used when an image cannot be displayed. This could be read by a screen reader, or displayed on screen due to an invalid src. Note: An alt attribute should be left blank if the image is purely visual, holding no relevance to the content displayed on the page, however it is still important to include an alt attribute for all <img> tags in order to adhere to standards.

<h1>, <h2>, <h3>, <h4>, <h5> & <h6>

Headings of varying sizes.

Headings should always contain the following:

  • id="" - Setting an ID will allow navigation to the header using anchor links.

<label>

Label tags must always contain the following:

  • for="" - The 'for' attribute on a label is used by screen readers to associate that label with a form control.

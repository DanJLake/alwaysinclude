alwaysInclude

Stadardised and Accessible HTML guide.

The aim of this guide is to list the HTML5 tags and their attributes that should be included in almost any HTML file, for either accessibility reasons, or to adhere to web standards.

<!DOCTYPE html>

The doctype declaration is required as a DTD (Document Type Definition). <!DOCTYPE html> refers to HTML5.

<html>

The <html> tag tells browsers this document is written in HTML format.

The <html> tag should contain the following:

  • lang="" - This attribute tells the browser the language the content of the page is written in. For example, 'en' would be the value for english, however this can be extended to 'en-gb' or 'en-us'.

The HTML tag is required in every HTML5 document.

<head>

Contains elements that do not display to the user. These can be links to stylesheets or scripts, meta tags, the page title, embedded styles or scripts, the page icon, and more.

The head element is required in every HTML5 document.

<body>

Contains all elements that will render or decide what is rendered.

All of a webpage's content is nested inside the body tag.

A body element is required in all HTML5 documents.

<link>

Links to external files such as CSS files and icons.

The <link> tag must always contain the following attributes:

  • href="" - the location of the file being linked. The file type extension must be included.
  • rel="" - specifies the contents of the document being linked. Possible values would be 'stylesheet' for a CSS stylesheet, or 'shortcut icon' for a favicon.

<script>

Script tags are used to wrap JavaScript imports or inline code.

The <script> tag should always contain the following:

  • src="" - The path to the file you are importing, if the script tag is being used to import a .js file.

<meta>

A <meta> tag is a multi-purpose element used to provide information to software that might be reading the file. This could be:

  • Browsers
  • Search Engines
  • Link Parsers
  • Social Networks
  • Many more

Meta tags are important for every HTML5 document for everything from displaying properly on mobile devices, to appearing at the top of search results.

Meta tags must always include:

  • name="" - Some software reading the file will only look for meta tags with a specific name value, setting this value correctly allows them to read the content.
  • content="" - The content of the meta tag. This differs between meta tags, as does its format.

All HTML5 documents should contain a meta tag with the charset="" attribute. The standard value being 'UTF-8'.

<header>

A <div> like container specifically for content placed at the top of a webpage, usually a top bar with a logo and navigation. A screen reader will recognise this as such allowing for greater accessibility.

<nav>

A container for the page navigation. Can be used by screen readers to quickly jump to the navigation links on a page. Usually placed inside the <header>, however it can be placed anywhere on the page.

<main>

The container that should hold the main content of a webpage. Screen readers will identify this content as the main content and allow the user to easily access it.

<article>

The container is used to wrap independent, self-contained content. Perfect for blog or news articles, or forum posts.

<footer>

The container that marks the page footer, this usually contains secondary navigation links, legal information, social media icons and other content that may be generalised site-wide, or not directly related to the content in the <main> section.

<section>

A container used to group related content. If none of the above HTML5 containers describe the section, or there are many sections on the page, this container allows screen readers to recognise the content inside is related.

A section tag should always contain the following:

  • id="" - An ID attribute identifies the nature and purpose of the section, whether vague or specific.

<figure>

The figure tag is used to wrap self-contained content related to the main content of the page. A figure tag is usually used with an image, however it can be used for a variety of media, including videos, code snippets, and more.

The figure tag works with the figcaption tag to link the contained content with it's descriptive caption. This is useful for screen readers and other assistive technology.

<figcaption>

The figcaption tag can be nested inside the <figure> tag to provide a description of the content it contains. This provides the user a clear description (caption) of the content, usually brief.

<figcaption> tags are useful for assistive technologies where the content of the <figure> may not be displayed.

<img>

Image tags must always contain the following:

  • src="" - The src of an image is its source file. This can be a link to a file stored in any accessible location, whether its local (stored on the same server/in the same folder or directory as the html file), or located elsewhere online.
  • alt="" - The alt value of an image is used when an image cannot be displayed. This could be read by a screen reader, or displayed on screen due to an invalid src. Note: An alt attribute should be left blank if the image is purely visual, holding no relevance to the content displayed on the page, however it is still important to include an alt attribute for all <img> tags in order to adhere to standards.

<h1>, <h2>, <h3>, <h4>, <h5> & <h6>

Headings of varying sizes.

Headings should always contain the following:

  • id="" - Setting an ID will allow navigation to the header using anchor links.

<label>

Label tags must always contain the following:

  • for="" - The 'for' attribute on a label is used by screen readers to associate that label with a form control.

<fieldset>

Used to group related elements in a form. The <fieldset> can display a box around this group if required, however grouping elements also allows assistive technology to recognise the grouped items as related, and inform the user of this. This can provide context to these elements.

<time>

Informs the browser that the nested content is time-related. This tag's real power is in it's 'datetime' attribute.

The <time> tag should always contain the following:

  • datetime="" - Used to provide a machine-readable date-time when the content provides a human-readable version. This can be used by calendar application on smartphones, among other things, as well as assistive technologies.

<legend>

The legend element is used to provide a title or caption for the element the legend element is nested inside. This can be used in conjunction with the <figure>, <fieldset> and <details> elements.

<details>

Creates a small, collapsed but extendable box containing the text nested in the tag. Typically used for disclosures or other information that should be hidden from the user, unless they choose to view it.

A <details> element should always contain a nested <summary> element as its first child. It can also use the following attributes:

  • open="" - Sets the default state of the details box. The default value of this is 'false' which will load the page with the box collapsed, if a value of 'true' is given, the details box will be open upon page load, but can still be toggled to hidden.

<summary>

Provides a summary of the details given in a <details> element. This summary is the content that will display in the collapsed details element, with all other content only being visible when the box is opened. If left blank this will be automatically assigned as 'Details'.

id=""

The ID attribute allows a HTML element to be individually identified. This is useful for a few reasons.

HTML5 documents will direct anchor links based on the ID they reference.

CSS can be applied to an element using it's ID. CSS styles IDs using the '#' selector.

JavaScript can reference or change elements using their ID with the getElementById() function.

name=""

The name attribute is used in HTML5 for the following:

  • Naming an element - Giving certain elements a name is useful, form elements for example can have their input read using their name as reference, and maps use the name attribute to identify the map for use in the usemap="" attribute.
  • Meta elements use the name attribute to identify their content.

JavaScript can reference or change an element using it's name using the getElementsByName() function, returning an array of all elements with the queried name.

While in older HTML versions the name attribute directed anchor links to the correct place, it is no longer supported for use with the <a> tag in HTML5.

class=""

The class attribute's main use is to apply class-specific styles to elements and their children.

Classes are styled in CSS using the '.' selector.

JavaScript can create an array of elements of any particular class using the getElementsByClassName() function.
